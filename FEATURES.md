# 🤖 Полное описание бота — возможности, команды, идеи

---

## 📋 ЧТО УМЕЕТ БОТ

### Для покупателей

**Каталог и наличие**
- Бесконечная вложенность категорий (категории внутри категорий до любой глубины)
- Кнопка «📋 Наличие» — одним сообщением показывает все категории и товары с кликабельными ссылками
- Каждый товар/категория в «Наличии» — ссылка, при нажатии открывает карточку прямо в боте
- Кнопка «🔔 Уведомить о поступлении» на товарах с нулевым остатком — бот напишет как только товар появится
- Карточка товара: цена, остаток, инструкция, скидка с балансом

**Покупка**
- Выбор количества (1–N)
- Оптовая скидка: от 3 шт. — скидка 5% (настраивается для каждого товара отдельно)
- Применение купона на скидку (% или фиксированная сумма в $)
- Скидки не суммируются — применяется та что больше (купон или опт)
- Два способа оплаты: с баланса бота или Telegram Stars
- При оплате Stars — инвойс прямо в чате, деньги не покидают Telegram
- Автоматическая выдача товара (ссылки, текст, файлы, комплекты файлов)
- Номер заказа в формате XXXX-XXXX-XXXX виден покупателю и администратору
- Парсинг нумерованных текстовых товаров (выдаётся точный блок под номером)

**Товары с запросом данных (буст, верификация)**
- Покупатель вводит количество → бот просит нужные данные (текст настраивается)
- Данные отправляются всем администраторам с кнопками «Подтвердить / Отклонить»
- После подтверждения покупатель получает уведомление и предложение оставить отзыв
- При отклонении — средства возвращаются на баланс

**Баланс**
- Пополнение через CryptoBot (USDT, +3% комиссия делением на 0.97)
- Пополнение через Telegram Stars (конвертация по курсу)
- Пополнение через FreeKassa (рубли, webhook)
- Пополнение в рублях через владельца
- Автоматическое зачисление (scheduler проверяет каждые 30 сек)
- Баланс применяется как скидка при покупке

**Купоны**
- Процентные (10% скидка)
- Фиксированные (5$ скидка)
- Каждый купон одноразовый для одного пользователя
- Приоритет: применяется наибольшая скидка (купон или оптовая), они не суммируются

**Реферальная система**
- Уникальная ссылка в профиле: `t.me/бот?start=ID`
- 5% с каждого пополнения баланса реферала (CB, Stars, FK)
- Начисляется мгновенно, реферер получает уведомление
- Статистика: количество рефералов и суммарный заработок в профиле

**Профиль (ЛК)**
- Имя, ID, баланс, заработок с рефералов
- Реферальная ссылка
- История заказов с пагинацией (по 5 шт., кнопки ◀️ ▶️)
- Статус каждого заказа: ⏳ ожидает / ✅ выполнен / ❌ отменён
- Код каждого заказа виден в истории

**Inline-кнопки под приветствием**
- 🛒 Каталог
- 📋 Наличие
- 💰 Пополнить баланс
- 👤 Профиль
- 🆘 Поддержка (ссылка)

### Для администраторов

**Управление каталогом**
- Создание категорий и подкатегорий любой глубины
- Удаление пустых категорий
- Создание товаров с выбором категории
- Описание товара (или «-» чтобы пропустить)
- Настройка оптовых условий для каждого товара (мин. кол-во и % скидки)
- Флаг «требует данных» с настраиваемым текстом запроса
- Добавление инструкции (текст и/или файл)
- Добавление товара (ссылки, текст нумерованным списком, файлы)
- Файловые товары с группами: несколько файлов = одна единица товара
  (бот спрашивает «ещё файл к этой единице?» после каждого)

**Пополнение остатков**
- Пополнить любой товар (активный или черновик) в любой момент
- Ссылки/текст списком или файлы
- Черновики (товары с 0 остатков) — отдельный раздел, активируются автоматически при добавлении

**Купоны**
- Создание купонов двух типов: % и $ (фиксированная сумма)
- Лимит активаций (0 = безлимит)
- Срок действия в днях (0 = бессрочно)
- Список всех купонов с использованием и сроком
- Удаление купонов

**Умная рассылка**
- Всем пользователям
- Только тем у кого баланс ≥ N$
- Тем кто покупал конкретный товар за последний месяц
- Тем кто покупал конкретный товар за всё время
- Чистый текст без дополнительных пометок, отчёт отправлено/ошибок

**Заказы**
- Раздел «Ожидающие заказы» — все заказы со статусом waiting_confirm
- Кнопки «Подтвердить» и «Отклонить» прямо в уведомлении
- Уведомление содержит: код заказа, пользователь, товар, сумма, данные

**Финансы**
- Начисление/списание баланса любому пользователю по Telegram ID
- Пользователь получает уведомление об изменении баланса
- Статистика: пользователей сегодня/всего, выручка (крипто отдельно), рефералам, заказов

**Мониторинг**
- Уведомление всем администраторам при остатке ≤ 3 шт. (настраивается)
- Уведомление при каждой покупке и пополнении
- Лог действий: последние 20 записей (создание товаров, покупки, пополнения, рассылки)
- Полный audit_log в БД (PostgreSQL, поиск по user_id, action, дате)

**Резервное копирование**
- Автоматический pg_dump каждые 6 часов
- SQL-файл отправляется всем администраторам в Telegram
- Хранятся последние 10 бэкапов

**Автопроверка платежей**
- Каждые 30 секунд проверяет все pending CryptoBot-платежи
- Зачисляет автоматически без нажатия кнопки пользователем
- Реферальный бонус начисляется тоже автоматически

## 💡 ЧТО ЕЩЁ МОЖНО ДОБАВИТЬ

### Нужно доработать (баги/ограничения текущей версии)

1. **Пагинация в каталоге** — если в категории > 50 товаров, inline-клавиатура не умещается. Нужна постраничная навигация в самом каталоге.

2. **Webhook вместо polling** — polling создаёт задержки при высокой нагрузке. Для 1000+ DAU лучше перейти на aiogram webhook mode + nginx.

3. **Graceful shutdown** — при перезапуске бота незавершённые FSM-сессии теряются. Нужен Redis-storage вместо MemoryStorage.

4. **Валидация FreeKassa курса** — сейчас курс $/₽ захардкожен (90). Нужно тянуть актуальный курс с ЦБ или binance API.

5. **Таймаут FSM-сессий** — если пользователь начал покупку и не завершил, сессия висит вечно. Нужен TTL на состояния.

6. **Транзакционность Stars-оплаты** — сейчас при ошибке выдачи Stars уже списаны. Нужно писать в pending-заказ до выдачи.

### Новый функционал (высокий приоритет)

7. **Система отзывов внутри бота** — через 15 минут после покупки бот сам присылает «оцените товар ⭐1-5». Средний рейтинг отображается в карточке.

8. **Тикеты поддержки** — кнопка «Создать обращение», переписка форвардится в группу. Ответы идут обратно через бота. Профессиональный уровень сервиса.

9. **Подписки / автопродление** — тип товара «подписка», бот помнит дату и за 3 дня до истечения присылает «продлить?» с кнопкой.

10. **Inline-режим** — `@бот claude pro` в любом чате выводит карточки товаров. Органический трафик.

11. **Уведомление о поступлении** — уже хранится в БД (таблица stock_notify), осталось добавить вызов в scheduler: при добавлении items проверять notify-список и рассылать.

12. **Реферальные уровни 2** — пригласил Васю, Вася пригласил Петю — ты получаешь 2% с Пети, Вася 5%.

13. **Промокоды для конкретного пользователя** — при создании купона указать user_id: только этот пользователь сможет активировать.

14. **Экспорт заказов** — кнопка «Экспорт CSV» в статистике: выгружает все заказы за период в файл.

15. **Мини-CRM** — поиск пользователя по username или ID в админке, просмотр его профиля: баланс, все заказы, рефералы.

16. **A/B тестирование рассылок** — отправить разные тексты двум группам, сравнить конверсию (клики на товары после рассылки).

17. **Автоматические купоны** — при первой покупке реферала автоматически выдавать рефереру купон (настраиваемый).

18. **Оплата TON** — CryptoBot поддерживает TON, ETH, BTC. Добавить выбор валюты при оплате.

19. **Подтверждение email/телефона** — для B2B клиентов запрашивать контакт при первой покупке и хранить в профиле.

20. **Интеграция с ботом уведомлений** — отдельный бот-канал куда бот форвардит все покупки красиво (для публичной демонстрации активности).

## ⚙️ НАСТРОЙКА PostgreSQL (быстрый старт)

```bash
# Установка на Ubuntu
apt install postgresql -y
sudo -u postgres psql

# В psql:
CREATE USER shopbot WITH PASSWORD 'yourpassword';
CREATE DATABASE shopbot OWNER shopbot;
\q

# В config.py:
DB_DSN = "postgresql://shopbot:yourpassword@localhost:5432/shopbot"
```

Схема создаётся автоматически при первом запуске (`run_migrations`).
